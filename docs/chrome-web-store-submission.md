# Chrome Web Store Submission Pack

This document is the working submission pack for the first FeedLens Chrome Web Store listing. Keep real API keys, private reviewer credentials, and developer-account details out of this repository.

## Owner Inputs Needed

Fill these in directly in the Chrome Web Store Developer Dashboard, not in committed files:

- Developer account email
- Publisher display name
- Public support email or support URL
- Privacy policy URL, for example the public `PRIVACY.md` URL after this repository is public
- Launch visibility: start with `Private` trusted testers or `Unlisted`, then move to `Public` after review confidence
- Temporary reviewer-only Gemini API key, if you choose to provide one in the private Test instructions field

Do not commit the temporary reviewer Gemini key. Do not include it in screenshots.

## Package

- Extension name: `FeedLens`
- Manifest version: `3`
- Extension version: `0.1.2`
- Production ZIP target: `release/feedlens-chrome-web-store-0.1.2.zip`
- Build source: production `dist/` generated by `npm run build`

Production package commands:

```sh
npm ci
npm run verify
test ! -e dist/debug.html && test ! -e dist/assets/debug.js
mkdir -p release
cd dist && zip -r ../release/feedlens-chrome-web-store-0.1.2.zip .
cd ..
npm run build:dev
```

The final `npm run build:dev` restores local debug-enabled `dist/` for browser testing after the production ZIP is created. Never upload the debug-enabled `dist/` output to the Chrome Web Store.

## Privacy Tab

### Single Purpose

```text
FeedLens analyzes visible LinkedIn and supported X timeline posts with the user's Gemini API key and shows local information-quality and manipulation-risk markers.
```

### Data Handling Summary

Declare that FeedLens handles user data because it processes visible website content, supported page URLs, a user-provided Gemini API key, settings, and local analysis results.

Suggested data categories to disclose when available in the dashboard:

- Website content: visible LinkedIn and X post text used for analysis
- Web browsing activity: supported-page URLs and current active tab context used to determine whether FeedLens can run
- Authentication information: the user-provided Gemini API key
- User-generated content: visible social post text may include user-generated content

Use the dashboard's exact category names if they differ.

### Limited Use Certification

Certify that:

- Data is used only for the user-facing FeedLens analysis and marker features.
- Data is not sold.
- Data is not used for advertising.
- Data is not transferred to the project creator.
- Visible post text is sent directly from the user's browser to Gemini only after the user configures a Gemini API key and accepts the privacy notice.
- Local analysis cache is stored in Chrome extension storage on the user's device.

## Permission Justifications

`storage`:

```text
Stores the user's Gemini API key, privacy notice acceptance, FeedLens settings, API key health status, and local analysis cache in Chrome extension storage on the user's device.
```

`activeTab`:

```text
Allows the popup to communicate with the currently active supported LinkedIn or X tab so the user can view status, analyze visible posts, pause or resume FeedLens, and clear visible markers.
```

`https://www.linkedin.com/*`:

```text
Runs the content script on LinkedIn pages so FeedLens can detect visible post containers, extract visible post text, and render local markers and Lens details.
```

`https://x.com/*`:

```text
Runs the content script on X pages so FeedLens can detect visible posts on supported home and profile timelines, extract visible post text, and render local markers and Lens details. Unsupported X routes are detected but not analyzed.
```

`https://generativelanguage.googleapis.com/*`:

```text
Allows the background service worker to send visible post text directly from the user's browser to Gemini using the user's configured API key.
```

## Test Instructions

Paste this into the dashboard's private Test instructions field and replace bracketed placeholders only inside the dashboard:

```text
FeedLens does not require a FeedLens account or backend login.

Setup:
1. Install the extension.
2. Open the FeedLens settings page.
3. Paste the temporary reviewer Gemini API key: [PASTE TEMPORARY REVIEWER GEMINI KEY HERE IN THE DASHBOARD ONLY]
4. Check the privacy notice consent box.
5. Save settings and confirm the Gemini check passes.

LinkedIn test:
1. Sign in to any LinkedIn account available to the reviewer.
2. Visit https://www.linkedin.com/feed/.
3. Open the FeedLens popup and click Analyze visible.
4. Confirm visible posts receive calm green, yellow, or red Lens markers.
5. Click a Lens marker and confirm inline details show scores, detected signals, confidence, counter-reading, and suggested action.
6. Use Pause and Clear markers from the popup.

X test:
1. Sign in to any X account available to the reviewer.
2. Visit https://x.com/home or a profile timeline such as https://x.com/{handle}.
3. Open the FeedLens popup and click Analyze visible.
4. Confirm visible posts receive Lens markers and inline details.
5. Confirm unsupported routes such as messages, search, notifications, and individual post detail pages do not auto-analyze.

Privacy and behavior notes:
- FeedLens analyzes only visible supported-platform posts.
- FeedLens does not auto-scroll.
- FeedLens does not like, comment, repost, follow, connect, message, or post.
- Content scripts never read the Gemini API key.
- FeedLens does not use a creator-operated backend or analytics.
- Gemini requests are sent directly from the browser to https://generativelanguage.googleapis.com/.
```

If you do not want to provide a reviewer Gemini key, replace step 3 with:

```text
Use any valid Gemini API key available to the reviewer.
```

Providing a temporary key usually makes review smoother. Keep that key restricted, monitor it, and revoke it after review.

## Pre-Submit Checklist

Run before uploading:

```sh
npm run verify
test ! -e dist/debug.html && test ! -e dist/assets/debug.js
rg "GEMINI_API_KEY|AIza" dist || true
git diff --check
find . -maxdepth 2 -type l -print -exec readlink {} \;
git status --short
```

Manual checks:

- Store description, privacy tab, `PRIVACY.md`, popup, and options page all describe the same data handling.
- Production ZIP root contains `manifest.json`, not a parent `dist/` folder.
- ZIP does not contain debug assets.
- ZIP does not contain `.env`, API keys, real user data, or private screenshots.
- Screenshots do not expose real social accounts, private posts, API keys, or browser profile details.
- Version in `public/manifest.json` is greater than any previously uploaded version.
- Launch visibility is intentionally selected: `Private`, `Unlisted`, or `Public`.
